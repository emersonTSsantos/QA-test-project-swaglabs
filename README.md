# 🧪 QA Manual Test Project -- Swag Labs

Este projeto tem como objetivo demonstrar, de forma prática, o processo
completo de **teste manual de software**, simulando o dia a dia de um
profissional de **Quality Assurance (QA)**.

O projeto foi desenvolvido utilizando a aplicação **Swag Labs**, uma
plataforma de e-commerce amplamente utilizada para prática de testes de
software.

Durante o projeto foram aplicadas diversas práticas utilizadas em
ambientes reais de desenvolvimento, incluindo:

-   Análise de sistema
-   Levantamento de requisitos
-   Criação de User Stories
-   Modelagem de cenários de teste
-   Mind Map de testes
-   Escrita de casos de teste
-   Execução de testes
-   Registro de bugs encontrados

Este repositório simula um **ciclo completo de QA manual**, desde a
análise inicial até a documentação de defeitos.

------------------------------------------------------------------------

Demonstrar habilidades práticas em **Quality Assurance**, incluindo:

-   Análise funcional de sistemas

-   Identificação e documentação de defeitos

Este projeto foi desenvolvido como parte da construção de **portfólio
profissional para atuação na área de Testes de Software**.

# 🧩 Sistema Testado

Sistema utilizado para os testes:

Swag Labs\
https://www.saucedemo.com/

permitindo testar funcionalidades como:

-   Login de usuários
-   Visualização de produtos
-   Adição de itens ao carrinho
-   Checkout

# 📋 Análise do Sistema

Durante a fase inicial do projeto foi realizada uma **análise das
principais funcionalidades do sistema**, identificando:

-   Fluxo de autenticação
-   Listagem de produtos
-   Funcionalidade de carrinho
-   Processo de checkout
-   Ordenação de produtos

O objetivo dessa etapa foi compreender o comportamento da aplicação e
identificar possíveis áreas críticas para testes.

Documento disponível em:

    docs/analise-do-sistema.md

# 👤 Histórias de Usuário (User Stories)

Foram criadas histórias de usuário baseadas nas principais
funcionalidades do sistema, utilizando a estrutura padrão:

    Como [tipo de usuário]
    Quero [ação]
    Para que [benefício]

Cada história inclui:

-   Descrição da funcionalidade
-   Regras de negócio
-   Critérios de aceitação

Documento disponível em:

    docs/historias-de-usuario.md

# 🧪 Casos de Teste

Foram elaborados **20 casos de teste**, utilizando dois formatos
amplamente utilizados na indústria de testes.

## Casos de Teste BDD

Criados utilizando a sintaxe **Gherkin**:

    Given
    When
    Then

Este formato é amplamente utilizado em práticas de **Behavior Driven
Development (BDD)**.

Arquivo:

    tests/casos-de-teste-bdd.md

Total de cenários:

10 casos de teste

------------------------------------------------------------------------

## Casos de Teste Step-by-Step

Casos de teste escritos no formato tradicional de QA, contendo:

-   ID do teste
-   Pré-condições
-   Passo a passo
-   Resultado esperado

Arquivo:

    tests/casos-de-teste-step-by-step.md

Total de cenários:

10 casos de teste
